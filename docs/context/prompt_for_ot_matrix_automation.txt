Нужно доработать userscript для Tampermonkey под страницу матрицы согласования OpenText.

Контекст:
- Это страница матрицы согласования OpenText.
- В таблице есть #sc_ApprovalMatrix.
- Каждая строка матрицы — tr[itemid] внутри #sc_ApprovalMatrix tbody.
- В строке есть hover-actions: new / copy / edit / save / rollback / delete.
- У OpenText на странице уже есть глобальный объект sc_ApprovalMatrix и jQuery.
- У sc_ApprovalMatrix есть рабочие внутренние методы toggleItemRenderState(jQueryRow, dontSave), saveItemDataFromView(...), renderPartnerPane(...), deletePartner(...).
- Поле контрагента рендерится как td.attrAlias_partner_id, а иногда как td.attrAlias_partners_internal_id.
- В view-mode там ссылки a.sc_partner.
- В edit-mode там hidden input с id-шниками контрагентов и ul/li.token-input-token со span.token-input-delete-token для удаления конкретного контрагента из строки.
- Вторая ячейка этого же атрибута содержит условие: «Использовать» / «Исключить».

Что уже есть:
- Базовый userscript, который умеет:
  1) показывать ч/б минималистичную панель;
  2) искать строки по названию контрагента;
  3) переводить строку в edit-mode;
  4) удалять нужный токен контрагента внутри строки;
  5) сохранять строку обратно.

Что нужно сделать теперь:
1. Добавить полноценную автоматизацию фильтра именно по первой колонке «Контрагент», а не только работу по видимым строкам.
2. Скрипт должен уметь:
   - открыть фильтр колонки «Контрагент»;
   - найти значение по названию контрагента;
   - отметить нужный checkbox;
   - нажать Apply;
   - после этого пройти только по отфильтрованным строкам.
3. Добавить режимы:
   - «Убрать контрагента из строки»;
   - «Удалить строку целиком», если контрагент в строке единственный.
4. Добавить dry-run режим:
   - ничего не менять,
   - только показать, какие строки будут затронуты,
   - указать itemid строки,
   - указать: будет удалён токен или будет удалена вся строка.
5. Добавить экспорт отчёта в JSON / CSV:
   - itemid,
   - исходный список контрагентов,
   - удалённый контрагент,
   - тип действия,
   - статус (ok / skipped / error).
6. Добавить safe-guards:
   - не трогать строку, если после удаления она станет слишком широкой по смыслу;
   - спрашивать отдельное подтверждение перед массовым удалением строк целиком;
   - логировать все ошибки в панели.
7. Сделать код аккуратным:
   - без мусора;
   - с маленькими функциями;
   - с понятными именами;
   - с комментариями только там, где реально нужно;
   - в современном vanilla JS, но можно использовать jQuery страницы там, где это реально помогает.

Требования к ответу:
- Отдай полностью готовый userscript целиком.
- Не пиши общую теорию.
- Сразу дай код.
- После кода коротко объясни, какие места завязаны на внутренний OpenText DOM и что, скорее всего, придётся подправить, если на другой матрице будут другие alias полей.
