# Политика безопасности

## Сообщение о уязвимостях

Не публикуйте уязвимость в открытом issue до исправления.  
Отправляйте детали maintainers в приватном канале, приложив:

- сценарий воспроизведения;
- влияние на данные/процессы;
- предполагаемый вектор атаки.

## Область риска

- операции apply в боевой матрице;
- доступ к данным контрагентов и согласующих;
- экспортные файлы и буфер обмена;
- live-профили тестирования через env-переменные.

## Базовые меры

- применяйте изменения только после превью;
- держите включенными draft-guard и лимиты затронутых строк;
- не храните credentials в репозитории.
